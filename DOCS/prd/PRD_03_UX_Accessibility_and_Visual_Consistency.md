# PRD 03: UX 卓越：无障碍合规与视觉一致性标准

## 1. 文档信息
- **版本**: v1.1
- **状态**: 已发布
- **最后更新**: 2026-02-04

## 2. 需求背景
随着项目规模扩大，代码的健壮性和视觉的精细度成为重点。需要修复控制台的 Accessibility (a11y) 报错，并解决因动态内容长度不一导致的界面参差不齐问题。

## 3. 功能需求

### 3.1 无障碍访问 (Accessibility) 标准化
- **弹窗要求**: 所有的 `DialogContent` 和 `SheetContent` 必须包含 `DialogTitle` 和 `DialogDescription`。
- **视觉处理**: 对于 UI 设计上不需要显示标题的弹窗（如照片预览、移动端菜单），使用 Tailwind 的 `sr-only` 类将其在视觉上隐藏，但保留在 DOM 中供屏幕阅读器使用。
- **范围**: 涵盖家长端照片预览、Boss 端学生/教师/班级管理、官网内容管理、移动端侧边栏。

### 3.2 课程卡片视觉对齐
- **动态字号**: 针对 `courses` 卡片的 `h3` 标题。
    - 字符数 > 10: `text-base`
    - 字符数 > 7: `text-lg`
    - 字符数 <= 7: `text-xl`
- **目标**: 确保长标题课程（如：Scratch图形化编程）不会因为换行而导致卡片高度高于其他卡片，保持网格布局的严丝合缝。

### 3.3 管理端触控优化
- **Tabs 触发器**: 增大 Boss 后台和官网管理后台的 Tabs 字体和图标大小。
- **规范**: 移动端使用 `text-sm sm:text-base`，图标统一调整为 `w-4 h-4`，增加点击热区。

## 4. 验收标准
- [ ] 浏览器控制台不再出现 `DialogContent requires a DialogTitle` 的警告。
- [ ] 课程乐园中的所有卡片高度在任何分辨率下均保持一致。
- [ ] 后台管理菜单在手机上易于手指点击，无误触。
