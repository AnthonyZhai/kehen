# PRD 02: 多端响应式导航与布局适配

## 1. 文档信息
- **版本**: v1.1
- **状态**: 已发布
- **最后更新**: 2026-02-04

## 2. 需求背景
系统需支持从 375px (iPhone SE) 到 1920px (Desktop) 的全维度设备访问。特别是针对 iPad 系列设备，需要在“移动端抽屉菜单”与“桌面端导航条”之间寻找最佳平衡点，防止文字溢出、重叠或垂直换行。

## 3. 详细适配策略

### 3.1 导航栏响应式断点逻辑
- **移动端 (宽度 < 768px)**: 
    - 使用汉堡菜单 (Sheet)。
    - **iPhone SE 优化**: 压缩菜单项垂直间距 (gap-8 -> gap-4)，减小按钮高度，确保“登录”和“管理”按钮不超出底部边界。
- **平板端 (768px <= 宽度 < 1280px)**: 
    - 针对 iPad Mini、iPad、iPad Pro。
    - **核心逻辑**: 显示横向导航条，但为了节省空间，使用 **2字简称**（如：关于、课程、荣誉、导师）。
    - **防换行**: 强制使用 `whitespace-nowrap`，防止标题（如“轻近编程”）和菜单项在空间不足时垂直排列。
- **桌面端 (宽度 >= 1280px)**: 
    - 显示横向导航条，使用 **4字全称**（如：关于我们、课程乐园）。

### 3.2 管理后台 (Boss/官网管理) 适配
- **Tabs 布局**: 在移动端（< 640px）下，将原本的“左右拖动/裁剪”模式改为“自动换行”模式。
- **布局容器**: 移除 `overflow-x-auto`，启用 `flex-wrap`，让按钮根据屏幕宽度自动排列。

## 4. 界面交互要求
- **标题保护**: 网站 Logo 标题“轻近编程”在任何设备上都必须横向显示，禁止折行。
- **iPad Pro 特殊处理**: 在 1024px 宽度下，虽然空间较大，但仍优先保证四字全称不换行，若空间仍紧迫则保持简称。

## 5. 验收标准
- [ ] iPhone SE 下菜单内容无需滚动即可看全底部按钮。
- [ ] iPad Mini 下标题和菜单不发生垂直错位。
- [ ] iPad Pro 下菜单名称显示完整且整齐。
- [ ] 后台管理菜单在手机上自动折叠成多行。
