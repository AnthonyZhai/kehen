# PRD 01: 官网内容动态化与试听系统

## 1. 文档信息
- **版本**: v1.1
- **状态**: 已发布
- **最后更新**: 2026-02-04

## 2. 需求背景
目前官网的课程内容和试听页面的课程选项是静态硬编码的。为了提高管理效率，确保“官网内容管理”后台修改后，前端展示和报名入口能同步更新，需要将数据流完全动态化。

## 3. 功能需求

### 3.1 数据库同步机制
- **数据源**: Supabase 数据库 `public_courses` 表。
- **字段要求**: 必须包含课程名称 (`name`)、适用年龄 (`age`)、描述 (`description`)、图标 (`icon_name`)。

### 3.2 预约试听页动态化
- **功能描述**: 试听页面的“意向课程”下拉菜单必须实时读取数据库内容。
- **展示逻辑**: 
    - 下拉选项格式：“课程名称 (适用年龄)”，例如：`Scratch图形化编程 (6-10岁)`。
- **技术实现**: 使用 `@tanstack/react-query` 的 `useQuery` 进行数据拉取，确保在后台更新课程后，用户刷新页面即可看到最新选项。

### 3.3 后台管理联动
- **官网内容管理**: 提供课程的增删改查功能。
- **一致性**: 删除某个课程后，试听页面的下拉选项应同步消失。

## 4. 验收标准
- [ ] 试听页面下拉菜单不再包含代码中写死的硬编码文字。
- [ ] 在后台添加一个新课程，试听页面刷新后能立即看到。
- [ ] 页面加载时有优雅的 Loading 状态或默认选项占位。
