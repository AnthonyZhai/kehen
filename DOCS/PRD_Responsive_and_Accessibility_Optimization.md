# PRD: 响应式布局与无障碍体验优化

## 1. 文档概览
本文档详细记录了针对“轻近编程”官网及其管理后台进行的响应式布局优化、无障碍访问修复以及动态内容对接的功能需求与实现。

## 2. 背景与目标
为了提升移动端和 iPad 端用户的访问体验，解决界面在不同尺寸屏幕下的布局错乱问题，并确保系统符合现代 Web 的无障碍访问标准（Accessibility）。

## 3. 主要功能需求

### 3.1 预约试听流程优化
- **动态课程加载**：预约试听页面的课程下拉菜单不再使用硬编码，而是实时从数据库的 `public_courses` 表中获取课程名称和适用年龄，确保官网展示内容与报名入口的一致性。

### 3.2 无障碍访问 (Accessibility) 修复
- **Dialog/Sheet 规范化**：为系统中所有的 `Dialog` 和 `Sheet` 组件（包括照片预览、学生管理、教师管理、移动端菜单等）添加隐藏的 `DialogTitle` 和 `DialogDescription`。
- **目标**：消除控制台无障碍访问警告，确保屏幕阅读器用户能够正确理解弹窗内容。

### 3.3 响应式布局优化 (移动端/iPad)
- **导航栏断点调整**：将桌面导航与移动端菜单的切换临界点优化为 `md` (768px)，并针对不同屏幕尺寸进行精细化适配。
- **自适应菜单标签**：
    - **平板端 (iPad/iPad Mini)**：显示两字简称（如“关于”、“课程”），防止导航项过多导致挤压。
    - **桌面端**：显示四字全称（如“关于我们”、“课程乐园”）。
    - **iPad Pro**：在 1024px 宽度下显示四字全称，通过 `whitespace-nowrap` 确保不换行。
- **移动端菜单优化**：
    - 针对 iPhone SE 等小屏手机，压缩垂直间距和字体大小，确保所有按钮（登录、管理系统）在不滚动的情况下可见。
    - 增加 `overflow-y-auto` 确保在极小屏幕下内容可滚动。
- **管理后台 Tabs 优化**：
    - 将 Boss 后台和官网管理后台的标签页（Tabs）从“水平滚动”改为“自动换行”模式。
    - 在移动端增大标签字体和图标大小，提升触控体验。

### 3.4 界面一致性与细节修复
- **课程卡片对齐**：根据课程标题长度动态调整 `h3` 标题的字号，确保标题始终保持在一行且不折行，维持卡片高度的一致性。
- **防挤压处理**：为 Logo 标题和导航文字添加 `whitespace-nowrap`，防止在临界尺寸下发生文字垂直排列。

## 4. 技术实现要点
- 使用 `@tanstack/react-query` 进行数据获取。
- 利用 Tailwind CSS 的响应式断点（`sm`, `md`, `lg`, `xl`）和 `sr-only` 类。
- 修改 `Index.tsx`, `Trial.tsx`, `BossDashboard.tsx`, `WebsiteContentManager.tsx` 等核心页面组件。
