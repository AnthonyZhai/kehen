# PRD 01: 系统核心业务逻辑与功能全景

## 1. 文档信息
- **版本**: v1.2 (全量功能补充版)
- **状态**: 已发布
- **最后更新**: 2026-02-04

## 2. 需求背景与项目简介
“课痕”是一款专为教育培训机构设计的智能化课时管理与家校互通系统。旨在解决教培机构课时统计混乱、签到繁琐、家校信息不对称等痛点，通过数字化手段连接老板、教师、家长三方，实现高效闭环管理。

## 3. 核心功能模块

### 3.1 教师端：智能考勤与签到 (📸)
- **核心逻辑**：教师上课时通过“一键拍照”发起签到。
- **自动化匹配**：系统根据拍照时间自动匹配当前时间段的班级。
- **自动扣费**：签到成功后，通过数据库触发器自动扣除学员对应课时。
- **可视化记录**：签到照片实时同步至家长端，作为上课凭证。

### 3.2 老板端 (Boss)：全能运营管理 (📊)
- **运营仪表盘**：实时展示总学员数、今日签到人数、续费预警名单。
- **学员/家长管理**：维护学员档案，管理课时账户，设置续费阈值。
- **教务管理**：管理教师账号、班级配置（含课时权重，如 1.0 或 1.5 课时/次）。
- **试听系统管理**：追踪官网试听申请状态（待处理、已联系、已完成）。
- **官网配置中心**：动态修改官网课程介绍、师资力量及轮播图。

### 3.3 家长端：透明化家校互通 (👥)
- **课时查询**：实时查看孩子的剩余课时及变动明细。
- **上课回顾**：查看教师上传的课堂照片和签到记录。
- **成就分享**：支持分享孩子的学习记录或成就卡片，提升机构口碑。

### 3.4 动态化对接 (专项优化)
- **课程同步**：官网展示的课程与预约试听页面的下拉选项实时同步。
- **数据源**：统一从 `public_courses` 表拉取，杜绝前端硬编码。

## 4. 业务流程要点
1. **招生阶段**：官网展示课程 -> 家长提交试听申请 -> Boss 后台管理申请。
2. **入读阶段**：Boss 录入学生 -> 绑定家长账号 -> 分配班级。
3. **日常教学**：教师拍照签到 -> 系统自动扣费 -> 家长接收通知/查看记录。
4. **存量阶段**：系统触发续费预警 -> 家长续费 -> 课时充值。

## 5. 验收标准 (系统级)
- [ ] 教师拍照签到后，学员课时必须实时扣除且准确无误。
- [ ] 家长登录后能立即看到最近一次上课的照片。
- [ ] Boss 后台修改课程信息后，官网及试听报名入口同步更新。
- [ ] 续费预警功能在课时不足时能正确标记。
